# Project Context & Guidelines

このドキュメントは、プロジェクトの進行における重要な決定事項、ユーザーの好み、制約事項を記録します。
開発を進める際は、常にこの内容を参照し、遵守してください。

## 1. Design Guidelines (デザイン方針)
*   **Tone & Manner**: 「信頼感」「プロフェッショナル」「堅実」。
    *   配色はネイビー、ブルー、ホワイトを基調とする。
    *   ダークモード寄りよりも、企業の信頼性を感じる白ベース＋濃紺のスタイルを採用（今回のLPで適用済み）。
*   **Assets**:
    *   **絵文字禁止**: 「ダサい」ため使用しない。代わりにSVGアイコンや高品質な画像を使用する。
    *   **Images**:
        *   **Style**: **フォトリアル（実写）**を基本とする。3Dイラストや抽象的なグラフィックは「チープ」に見える可能性があるため避ける。
        *   **Hero**: 横幅いっぱいの背景画像として配置し、没入感と信頼感を演出する。
    *   **Typography**: 読みやすく、モダンなフォント（Inter, Noto Sans JPなど）。

*   **ビジネスプラン**: **¥100,000**
